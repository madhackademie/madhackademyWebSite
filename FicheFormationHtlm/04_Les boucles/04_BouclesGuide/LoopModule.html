Les Boucles C++

GameDevReady

Objectif du Module

Les boucles sont l'un des piliers fondamentaux de la programmation. Elles permettent de répéter des actions sans avoir à réécrire le même code des centaines ou milliers de fois. Imaginez notre grenouille qui doit traverser plusieurs rangées de voitures : sans boucles, il faudrait écrire le code de vérification de collision pour chaque voiture individuellement !

Dans ce module, nous allons découvrir les quatre types principaux de boucles en C++ : for, range-based for (foreach), while et do-while. Chacune a ses spécificités et ses cas d'usage appropriés.

Nous allons pour la plupart étudier le même algorithme avec chaque boucle.

Nous aborderons également les dangers des boucles infinies et des boucles mal optimisées qui peuvent causer des ralentissements, des plantages. Ces problèmes sont particulièrement critiques dans le développement de jeux où la performance est essentielle.

Ce que l'élève saura faire à la fin

Utiliser les quatre types de boucles (for, range-based for, while, do-while) de manière appropriée, comprendre leurs différences, et identifier les pièges de performance à éviter.

Notions techniques abordées : Boucles for, while, do-while, conditions d'arrêt, optimisation, incrémentation et prévention des boucles infinies.

Théorie

La boucle WHILE : La condition persistante

La boucle while s'exécute tant qu'une condition reste vraie. C'est comme dire : "Continue à avancer tant que le chemin est libre". Vous l'avez déjà vue plein de fois sans le savoir.

La boucle DO-WHILE : L'action garantie

La boucle do-while exécute le code au moins une fois avant de vérifier la condition. C'est utile quand vous voulez faire quelque chose, puis décider si vous continuez.

int userChoice;
do {
    printf("Continuer le jeu ? (1=Oui, 0=Non) : ");
    scanf("%d", &userChoice); 
} while (userChoice == 1);

Je dois vous expliquer ce qu'est le scanf. Il s'agit de récupérer les données de l'utilisateur. En programmation il y a deux grands flux de données les entrants input et sortants output. Prenons à titre d'exemple un jeu vidéo (c'est d'ailleurs la raison de votre présence ici).

  • Une manette Nintendo, un clavier sont les input l'arrivée des données
  • L'écran où les données s'affichent est un output la sortie des données

Dans notre cas, nous avons déjà utilisé les flux sortants avec le printf et le DrawText. Pour afficher les données à l'écran que se soit dans la console ou sur l'écran.

La boucle FOR : Le compteur précis

La boucle for est idéale quand vous savez exactement combien de fois vous devez répéter une action. C’est comme monter un escalier dont on connaît le nombre de marches.

  • On commence à la marche 0 (ou celle que l'on a initialisée).
  • Tant qu'on n'a pas atteint la 10ème marche (condition), on continue.
  • À chaque mouvement, on monte d' une marche (incrémentation).

Structure d'une boucle for :

for (initialisation; condition; incrémentation) {
    // Code à répéter
}

L'exemple va illustrer 2 applications. Recréer le le compteur précédent avec cette boucle, et ensuite remplir un tableau (array) avec la boucle for .

Compter avec la boucle for

Cette fois-ci, le compteur est directement intégré à la boucle for, il y a l'instruction i qui jouerea le rôle de compteur.
Les instructions d'une boucle for servent à définir le comportement/fonctionnement de la boucle. A quel chiffre on commence, comment on avance ou recule et quand on s'arrête.
Petite subtilité la dernière instruction d'incrémentation n'est effectuée que quand le code entre les accolades (le block {}) est terminé. C'est pourquoi il y a un i + 1 dans le block {} au niveau du printf.
Et seulement après l'incrémentation la boucle recommence et vérifie si la condition est vraie ou fausse.
Il est possible d'initiliser le i avec la valeur de 1 qui rend le compteur plus simple, plus lisible, en bref plus propre.
Mais je tiens à vous habituer à la valeur de 0 qui est très courante à cause du parcour de tableau.
La première instruction elle n'est faite qu'une seule fois tout au début. Il s'agit d'une initialisation de variable utile uniquement dans la portée de la boucle for.

L'exemple suivant concerne la lecture/ecriture dans les conteneur ou collection (liste,tableau)

Affectation de valeur aux éléments d'un array avec la boucle for

On initialise un array que d'une taille de 5 éléments float vides. En gros un tableau avec 5 cases vides qui ne pourrons contenir que des float.

⚠️ Il est impératif que la condition ne dépasse pas la taille de l'array. Sinon, vous allez lire/écrire à un emplacement mémoire qui ne vous est pas attribué (votre mot de passe Ledger avec les 2k bitcoins).
J'ai besoin d'utiliser le i de la boucle for pour pouvoir me déplacer dans le tableau. Et je l'utilise pour calculer une valeur à affecter à cette emplacement de l'array.
On peut très bien mettre la même valeur dans chaque emplacement de l'array cela dépend encore une fois de votre objectif.
Et pour avoir un feedback j'ai tracé (print) les valeurs dans la console.
Nous allons survoler le code pour l'instant, l'objectif est de comprendre le fonctionnement de la boucle for.

Décomposition :

  • int i = 0 : Initialisation - On commence à compter à partir de 0
  • i < 5 (la taille de l'array) : Condition - On continue tant que i est inférieur à 5
  • i++ : Incrémentation - On ajoute 1 à i après chaque tour
Pour expliquer en quelques mots cette boucle for effectuons quelques itérations ensemble :
  • Le premier élément d'un tableau (array) est toujours à l'index 0
  • La boucle commence avec i = 0 du coup notre position dans l'array est 0 (la première case),
    et la valeur est = (0 + 1) * 0.62f
  • On vérifie la condition : i < 5 (est-ce que i est inférieur à 5 ?) → VRAI
  • i++ correspond à i = i + 1 soit 0 + 1 = 1 il s'agit d'une incrémentation
  • La position dans l'array est donc 1 qui correspond à la deuxième case. C'est un peu perturbant cette histoire de 0 et 1, mais voyez-le comme le fait de monter un immeuble avec le rez-de-chaussé qui a la valeur 0 mais qui est quand même un étage(case ou cellules) avec des appartements(data ou valeur).
    Ensuite vous gravissez un étage ce qui vous amène à la 2ème série d'appartements,mais vous n'êtes qu'au premier étage
  • Et on continue avec i=2 etc... tant que i est inférieur à 5

Ne vous prenez pas trop la tête avec le système de l'index et du i qui vaut 0 mais qui est 1. A force de gerer des boucles et des tableaux, ça rentrera tout seul. Le printf est votre ami ici, c'est pourquoi nous l'avons vue dès le début.

La boucle RANGE-BASED FOR (foreach) : L'itérateur moderne

La boucle range-based for (aussi appelée "foreach" dans d'autres langages) est la façon moderne et élégante de parcourir des conteneurs.
Toutefois, il est inutile de vouloir créer un compteur avec cette boucle c'est contre-productif et sans intérêt.
C'est comme dire : "Pour chaque élément dans la liste, fais quelque chose".
Un peu comme la boucle for précédente qui parcours un array. Sauf qu'ici on a pas besoin de créer un index la boucle gère toute seul le parcours des éléments.
Il y a une option pour ajouter un index ou compteur qui peut être utile dans certains cas de figure.
Nous y reviendrons dans le module sur les conteneurs. C'est pourquoi nous n'irons pas plus loin sur cette boucle qui sera vue en détails prochainement.
Je laisse quand même le code de base pour les curieux. 

// Parcourir un tableau de positions de voitures
int carPositions[] = {100, 250, 400, 550, 700};

// ✅ Syntaxe moderne et lisible
for (int position : carPositions) {
    printf("Voiture à la position : %d\n", position);
}

do-while Action garantie minimum 1 fois Condition vérifiée après
for Nombre d'itérations connu Compteur intégré
range-based for Parcourir un conteneur Moderne, simple, sûr

Les Dangers des Boucles

1. La Boucle Infinie : Le piège mortel 💀

🚨 DANGER CRITIQUE : Une boucle infinie va figer votre programme et consommer 100% d'un cœur de votre processeur. Dans le pire des cas, elle peut faire planter votre ordinateur !

Exemple de boucle infinie catastrophique :

// ❌ MAUVAIS - Boucle infinie !
int i = 0;
while (i < 10) {
    printf("Je vais tourner éternellement...\n");
    // Oups, j'ai oublié d'incrémenter i !
}
// ✅ BON - Condition d'arrêt garantie
int i = 0;
while (i < 10) {
    printf("Itération numéro %d\n", i);
    i++;  // N'oubliez JAMAIS l'incrémentation !
}

2. Les Boucles Imbriquées : Le monstre de performance 🐌

Plus vous imbriquez de boucles, plus la complexité explose exponentiellement. Une triple ou quadruple boucle peut rapidement devenir cauchemardesque.

Calcul de complexité :
  • 1 boucle de 1000 : 1 000 itérations
  • 2 boucles de 1000 : 1 000 000 itérations
  • 3 boucles de 1000 : 1 000 000 000 itérations (1 milliard !)
  • 4 boucles de 1000 : 1 000 000 000 000 itérations (1 trillion !!!)
// ❌ CATASTROPHIQUE - Ne faites JAMAIS ça !
for (int i = 0; i < 1000; i++) {
    for (int j = 0; j < 1000; j++) {
        for (int k = 0; k < 1000; k++) {
            for (int l = 0; l < 1000; l++) {
                // Calcul complexe ici
                double result = sin(i) * cos(j) * tan(k) * sqrt(l);
                // Votre jeu va geler pendant des heures !
            }
        }
    }
}

3. Les Opérations Lourdes dans les Boucles 🏋️

🚨 À ÉVITER ABSOLUMENT :
  • Charger des fichiers dans une boucle
  • Copier/coller de grandes structures de données
  • Faire des calculs mathématiques complexes sans nécessité
  • Allouer de la mémoire de manière répétée
// ❌ TRÈS MAUVAIS - Allocation mémoire répétée
for (int i = 0; i < 10000; i++) {
    char* bigArray[1000000] = {};  // 1 Mo à chaque tour !

}

// ✅ BIEN MIEUX - Allocation unique
char* bigArray[1000000] = {};  // 1 seule fois
for (int i = 0; i < 10000; i++) {
    // Réutilisation du même tableau

Résumé

À retenir

  • FOR classique : Quand vous connaissez le nombre exact d'itérations ou avez besoin de l'index
  • RANGE-BASED FOR : Syntaxe moderne et performante pour parcourir des conteneurs - À privilégier quand possible !
  • WHILE : Quand la condition peut être fausse dès le début (boucle peut ne jamais s'exécuter)
  • DO-WHILE : Quand vous voulez garantir au moins une exécution
  • DANGER : Toujours prévoir une condition d'arrêt pour éviter les boucles infinies
  • PERFORMANCE : Éviter les opérations lourdes et les boucles trop imbriquées (max 2-3 niveaux)
  • OPTIMISATION : Range-based for a des performances équivalentes ou meilleures que for classique avec un code plus lisible

Choix rapide de boucle :

  • 📊 Parcourir un tableau/vector ? → range-based for
  • 🔢 Besoin de l'index ou compteur ? → for classique
  • ⏳ Condition à vérifier d'abord ? → while
  • ✅ Exécuter au moins 1 fois ? → do-while

Définitions

Algorithme : Un mot un peu fourre-tout. Disons qu'il s'agit d'une chaîne ou suite d'opération pour arriver au but fixé. 1 dégager le bordel devant l'étagère pour y acceder. 2. et ensuite thématiser les éléments du bordel. 3.Les rangers dans l'étagère suivant le thème (ou la méthodologie de rangement choisie). Dans mon exemple ou a juste 1.Création d'un tableau 2. Remplissage du tableau. 3.Lecture du tableau et affichage selon le contenu.

Incrémentation : Il s'agit d'ajouter une unité à une variable numérique. Par exemple si j'ai une variable compteur qui vaut 0 et que je fais une incrémentation j'obtiens 1. Si je refais une incrémentation j'obtiens 2. C'est très utilisé dans les boucles pour compter le nombre d'itérations effectuées. Il y a aussi la décrémentation qui elle soustrait une unité à la variable numérique.Le principe de postfixé et préfixé est vue dans le lien fourni.

Conteneur : Il s'agit d'une collection regroupant des données. Vous avez déjà vu l'array mais ils sont dérivés en plusieurs formes. La difficulté réside dans le choix du type de conteneur que vous voulez ou devez utiliser. Certains se prêtent mieux à certaines manipulations que d'autres. Nous verrons ceux qui sont les plus courants et pratique dans la majorité des cas dans peu de temps. Mais si vous êtes curieux le lien fournit est très complet et je vous donne l' algorithme en image pour choisir un conteneur selon vos besoins.
