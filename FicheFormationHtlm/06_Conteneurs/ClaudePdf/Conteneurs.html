Les Conteneurs C++

GameDevReady

Objectif du Module

Imaginez notre grenouille qui doit traverser une route avec plusieurs voitures. Comment stocker toutes ces voitures en mémoire ? Comment gérer leurs positions, vitesses et couleurs ? C'est exactement à cela que servent les conteneurs !

Les conteneurs (ou collections) sont des structures de données qui permettent de stocker plusieurs éléments du même type. Au lieu de créer voiture1, voiture2, voiture3... nous pouvons créer un tableau voitures[] qui contient toutes nos voitures.

Dans ce module, nous allons découvrir deux types de conteneurs essentiels en C++ : les tableaux classiques (array) et les vectors (std::vector). Il y aura une présentation des tableaux dit clés/valeurs (dictionnaire ou map). Nous verrons également comment organiser nos données avec les structures (struct) et comment combiner tout cela ensemble.

Ce que l'élève saura faire à la fin

Créer et manipuler des tableaux et des vectors, comprendre le concept d'index, accéder aux éléments, parcourir des conteneurs, créer des structures, et accéder aux éléments d'une structure stockée dans un tableau.

Notions techniques abordées : Tableaux (array), vectors (std::vector), index, structures (struct), accès aux éléments, méthodes push_back(), pop_back(), size(), et parcours de conteneurs.

Déclaration d'une structure

struct Car {

    int y;         // Position Y

Créer et utiliser une structure


#include <cstdio>

struct Frog {
    int x;
    int y;
    int lives;
};

int main() {
    // Créer une grenouille
    Frog player;
    
    // Accéder aux membres avec le point (.)
    player.x = 400;
    player.y = 500;
    player.lives = 3;
    
    printf("Grenouille en position (%d, %d) avec %d vies\n", 
           player.x, player.y, player.lives);
    
    // Modifier un membre
    player.x += 50;  // La grenouille saute de 50 pixels
    player.lives--;  // Perd une vie
    
    printf("Nouvelle position : %d, Vies restantes : %d\n", 
           player.x, player.lives);
    
    return 0;
}

Théorie

Qu'est-ce qu'un index ? 📍

L'index (ou indice) est la position d'un élément dans un conteneur. C'est comme les numéros de places dans un train. Déjà vu dans le module précédent sur les boucles.
Mais une petite piqûre de rappel ne fait pas de mal.
C'est à force de répétition qu'il y a assimilation, n'oubliez pas notre devise (ou motto) et notre méthodologie.

⚠️ IMPORTANT : En programmation, l'index commence TOUJOURS à 0, pas à 1 !

  • Premier élément : index 0
  • Deuxième élément : index 1
  • Troisième élément : index 2
  • Et ainsi de suite...

Visualisation avec notre grenouille et des rondins :

// Rondins sur la rivière
//  Index:     0        1        2        3        4
//            [🪵]      [🪵]      [🪵]      [🪵]      [🪵]
// Position:  100      200      300      400      500

Les Tableaux Classiques (Array) 📦

Un tableau classique est une collection de taille fixe. Une fois créé, vous ne pouvez ni ajouter ni retirer d'éléments.

Syntaxe de déclaration :

type nomTableau[taille];

Exemple concret avec des positions de voitures :


#include <cstdio> 

int main() {
    // Déclaration d'un tableau de 5 positions de voitures
    int carPositions[5];
    
    // Initialisation (remplissage) des éléments
    carPositions[0] = 100;
    carPositions[1] = 250;
    carPositions[2] = 400;
    carPositions[3] = 550;
    carPositions[4] = 700;
    
    // Affichage d'un élément spécifique
    printf("La voiture à l'index 2 est en position : %d\n", carPositions[2]);
    
    return 0;
}

Déclaration avec initialisation directe :

// Plus rapide et lisible !
int carPositions[5] = {100, 250, 400, 550, 700};

Accéder à un élément du tableau 🎯

Pour accéder (lire ou modifier) un élément, on utilise la syntaxe suivante :

nomTableau[index]

Exemples pratiques :

int positions[3] = {100, 200, 300};

// Lire un élément
int premierElement = positions[0];  // Vaut 100
int deuxiemeElement = positions[1]; // Vaut 200

// Modifier un élément
positions[0] = 150;  // Change 100 en 150
positions[2] = 350;  // Change 300 en 350

printf("Nouvelle position : %d\n", positions[0]); // Affiche 150
⚠️Attention aux dépassements d'index !

Si votre tableau a 5 éléments (index 0 à 4) et que vous essayez d'accéder à tableau[5] ou tableau[10], vous accédez à une zone mémoire invalide ! Cela peut causer des bugs imprévisibles ou faire planter votre programme. 

int tableau[5];  // Index valides : 0, 1, 2, 3, 4
tableau[5] = 10; // ❌ ERREUR ! Index hors limites
tableau[0] = 10; // ✅ OK

Parcourir un tableau avec une boucle 🔄

Pour accéder à tous les éléments d'un tableau, on utilise généralement une boucle for :


#include <cstdio>

int main() {
    int carPositions[5] = {100, 250, 400, 550, 700};
    
    // Parcours avec for classique
    for (int i = 0; i < 5; i++) {
        printf("Voiture %d en position : %d\n", i, carPositions[i]);
    }
    
    return 0;
}

Avec la boucle range-based for (plus moderne) :

int carPositions[5] = {100, 250, 400, 550, 700};

// Parcours moderne (foreach)
for (int position : carPositions) {
    printf("Position : %d\n", position);
}

La Boucle Foreach (Range-Based For) 🔄

Maintenant que nous abordons les conteneurs, nous pouvons introduire la boucle foreach (appelée range-based for en C++). Cette boucle permet de parcourir facilement tous les éléments d'une collection

Syntaxe originale avec type explicite


#include <vector>
#include <cstdio>

int main() {
    int scores[4] = {100, 250, 180, 320};
    
    // Syntaxe : type item : collection
    for (int score : scores) {
        printf("Score : %d\n", score);
    }
    
    return 0;
}

Décomposition :

  • int : Type des éléments du conteneur
  • score : Variable temporaire qui prend la valeur de chaque élément
  • scores : Le conteneur à parcourir

Version moderne avec auto (recommandé) 🚀

Le mot-clé auto permet au compilateur de déduire automatiquement le type, rendant le code plus flexible et lisible. Si vous êtes amené à changer le type de la collection, vous ne devez pas changer toutes les boucles foreach qui parcourent le tableau. Cela évite des conversions implicites risquées et coûteuses en ressources au runtime.
(en clair le compilateur gère ceci avant le lancement du code et non pas pendant de l'exécution du code) 

int scores[4] = {100, 250, 180, 320};

// ✅ Version moderne avec auto
for (auto score : scores) {
    printf("Score : %d\n", score);
}

Modification des éléments avec la référence (&)

int scores[3] = {100, 250, 180};

// Doubler tous les scores
for (auto& score : scores) {
    score *= 2;  // Modification de l'élément original
}

// Affichage
for (auto score : scores) {
    printf("Nouveau score : %d\n", score);
}
// Affiche : 200, 500, 360
Différence importante :
  • auto : Copie de l'élément (lecture seule, pas de modification)
  • auto& : Référence à l'élément (permet la modification)
  • const auto& : Référence constante (lecture seule, mais pas de copie = plus performant)

Exemple avec structures


#include <cstdio>

struct Car {
    int x;
    int speed;
};

int main() {
    Car carsArray[] = {
        {100, 5},
        {300, 3},
        {500, 7}
    };
    
    // Lecture seule avec const auto&
    for (const auto& car : carsArray) {
        printf("Voiture en x=%d, vitesse=%d\n", car.x, car.speed);
    }
    
    // Modification avec auto&
    for (auto& car : carsArray) {
        car.x += car.speed;  // Déplacer la voiture
    }
    
    return 0;
}

Comparaison For classique vs Foreach

Critère For classique Foreach (auto)
Lisibilité Moins claire ✅ Plus simple
Accès index ✅ Oui ❌ Non
Risque erreur Possible (dépassement) ✅ Très faible
Performance Équivalente Équivalente
Cas d'usage Besoin de l'index Parcours simple

Quand utiliser quoi ?

  • 🎯 Foreach (auto) : Pour parcourir tous les éléments simplement
  • 🔢 For classique : Quand vous avez besoin de l'index ou d'un parcours complexe
  • ⚡ const auto& : Pour les structures volumineuses (évite la copie)
  • ✏️ auto& : Quand vous devez modifier les éléments

Les Références et Constantes 🔗

Maintenant que vous manipulez des conteneurs, il est temps de comprendre deux concepts essentiels : les références et les constantes. Ces notions sont particulièrement utiles avec les boucles foreach !

Les Constantes (const) 🔒

Une constante est une variable dont la valeur ne peut pas être modifiée après son initialisation. C'est comme un coffre-fort verrouillé !. La pratique voudrait que l'on définisse toutes les variables qui ne changent pas comme constante. Le principe d'immuabilité offre une sécurité et robustesse au code et évite les modifications accidentelles. Le nom de cette pratique est laConst-correctness. 


#include <cstdio>

int main() {
    const int MAX_LIVES = 3;
    
    printf("Vies maximum : %d\n", MAX_LIVES);
    
    // MAX_LIVES = 5;  // Ne compilera pas
    // ❌ ERREUR : impossible de modifier une constante !
    
    return 0;
}

Avantages des constantes :

  • 🛡️ Protection contre les modifications accidentelles
  • 📖 Rend le code plus lisible (on sait que la valeur ne changera pas)
  • ⚡ Optimisations possibles par le compilateur
  • 🐛 Détection d'erreurs à la compilation plutôt qu'à l'exécution

Les Références (&) 🔗

Une référence est un alias (un autre nom) pour une variable existante. C'est comme donner un surnom à quelqu'un : la personne reste la même, mais on peut l'appeler différemment. 


#include <cstdio>

int main() {
    int frogX = 100;
    
    // Créer une référence vers frogX
    int& positionX = frogX;
    
    // Modifier via la référence modifie aussi l'original !
    positionX = 200;
    
    printf("frogX = %d\n", frogX);        // Affiche : 200
    printf("positionX = %d\n", positionX); // Affiche : 200
    
    // Les deux variables pointent vers la MÊME case mémoire
    
    return 0;
}

Différence référence vs copie :

int original = 10;

// COPIE : crée une nouvelle variable indépendante
int copie = original;
copie = 20;
// original vaut toujours 10, copie vaut 20

// RÉFÉRENCE : crée un alias vers la même variable
int& reference = original;
reference = 30;
// original vaut 30, reference vaut 30 (même variable !)

Combinaison : const + référence (const &) 🔒🔗

On peut combiner les deux pour créer une référence constante : un alias qu'on ne peut pas modifier. C'est très utile pour éviter les copies tout en garantissant qu'on ne modifiera pas l'original.Et surtout très performant dès qu'on manipule de grosses données. Imaginer copier un gros atlas à chaque fois pour lire un élément/article en boucle. Il s'agirait d'un énorme gaspillage de ressources mémoire et un donc un gros ralentissement du programme devant attendre la fin de la copie.
Je suppose que sur une machine lambda subirait un crash du type "le programme ne répond plus ? attendre : annuler". 

struct Car {
    int x;
    int y;
    int speed;
};

int main() {
    Car myCar = {100, 200, 5};
    
    // Référence constante : pas de copie, mais lecture seule
    const Car& carRef = myCar;
    
    printf("Position : %d\n", carRef.x);  // ✅ OK : lecture
    
    // ❌ ERREUR : impossible de modifier via une référence constante
    // carRef.x = 150;  // Ne compilera pas
    
    return 0;
}

Utilisation dans les boucles foreach

C'est ici que tout prend son sens ! Voici les différentes façons d'utiliser foreach avec références et constantes :

Car cars[] = {
    {100, 150, 5},
    {300, 150, 3},
    {500, 150, 7}
};

// ❌ COPIE : chaque car est une copie (lent pour les grosses structures)
for (Car car : cars) {
    printf("x = %d\n", car.x);
}

// ✅ RÉFÉRENCE : pas de copie, on peut modifier
for (Car& car : cars) {
    car.x += 10;  // Modifie l'original dans le vector
}

// ✅✅ RÉFÉRENCE CONSTANTE : pas de copie, lecture seule (IDÉAL)
for (const Car& car : cars) {
    printf("x = %d\n", car.x);  // Lecture OK
    // car.x = 100;  // ❌ Erreur : modification interdite
}

Règle d'or pour les boucles foreach

  • 🐌 auto car : Copie l'élément (lent pour les structures volumineuses)
  • ✏️ auto& car : Référence, permet la modification
  • ⚡ const auto& car : Référence constante, lecture seule (MEILLEUR CHOIX par défaut)

Comparaison Performance

Syntaxe Copie ? Modification ? Performance Cas d'usage
auto item ✅ Oui ❌ Non (copie locale) 🐌 Lent Rarement utilisé
auto& item ❌ Non ✅ Oui ⚡ Rapide Modification nécessaire
const auto& item ❌ Non ❌ Non ⚡ Rapide ✅ Par défaut !

Exemple pratique complet


#include <cstdio>

struct Enemy {
    int health;
    int damage;
    bool isAlive;
};

int main() {
    Enemy enemies[] = {
        {100, 10, true},
        {150, 15, true},
        {80, 8, true}
    };
    
    // 1. Afficher les ennemis (const auto& = pas de copie, lecture seule)
    printf("=== ÉTAT INITIAL ===\n");
    for (const auto& enemy : enemies) {
        printf("HP: %d, Damage: %d\n", enemy.health, enemy.damage);
    }
    
    // 2. Infliger des dégâts (auto& = modification autorisée)
    printf("\n=== ATTAQUE ! ===\n");
    for (auto& enemy : enemies) {
        //Imaginer une attaque de zone sur tous les enemy. holy grenade
        enemy.health -= 50;
        if (enemy.health <= 0) {
            enemy.isAlive = false;
        }
    }
    
    // 3. Compter les survivants (const auto& = lecture optimisée)
    int aliveCount = 0;
    for (const auto& enemy : enemies) {
        if (enemy.isAlive) {
            aliveCount++;
        }
    }
    
    printf("Ennemis vivants : %d\n", aliveCount);
    
    return 0;
}
Pièges à éviter :
  • ❌ Oublier le & avec de grosses structures = copies inutiles et lentes
  • ❌ Modifier via auto& alors qu'on voulait juste lire = bugs potentiels
  • ✅ Toujours privilégier const auto& sauf si modification nécessaire

Les Vectors : Tableaux Dynamiques 🚀

Un std::vector est un tableau dynamique : sa taille peut changer pendant l'exécution du programme. Vous pouvez ajouter ou retirer des éléments à volonté ! Il sera l'un de vos principaux outil avec l'array.

Pour utiliser les vectors, il faut inclure :

#include <vector>

Déclaration et initialisation d'un vector


#include <vector> 
#include <cstdio> 

int main() {
    // Déclaration d'un vector vide
    std::vector<int> carPositions;
    
    // Déclaration avec initialisation
    std::vector<int> frogJumps = {10, 20, 30, 40, 50};
    
    return 0;
}

Méthodes essentielles des vectors 🛠️

Méthode Description Exemple
push_back() Ajouter un élément à la fin vec.push_back(100);
pop_back() Retirer le dernier élément vec.pop_back();
size() Obtenir le nombre d'éléments int n = vec.size();
empty() Vérifier si le vector est vide if (vec.empty())
clear() Vider complètement le vector vec.clear();

Exemple complet avec ajout/suppression


#include <vector>
#include <cstdio>

int main() {
    std::vector<int> carPositions;
    
    // Ajout d'éléments
    carPositions.push_back(100);
    carPositions.push_back(250);
    carPositions.push_back(400);
    
    printf("Nombre de voitures : %zu\n", carPositions.size());
    // Affiche : 3
    
    // Accès à un élément (comme avec les tableaux)
    printf("Première voiture : %d\n", carPositions[0]);
    // Affiche : 100
    
    // Modification d'un élément
    carPositions[1] = 300;
    
    // Suppression du dernier élément
    carPositions.pop_back();
    
    printf("Nouvelles positions :\n");
    for (int i = 0; i < carPositions.size(); i++) {
        printf("Voiture %d : position %d\n", i, carPositions[i]);
    }
    
    return 0;
}

Pourquoi %zu pour size() ?

La méthode size() retourne un type spécial (size_t) qui nécessite le format %zu avec printf. Vous pouvez aussi caster en int : (int)vec.size() et utiliser %d. Le cast et size_t seront vue ultérieurement dans la formation, lors de cas pratique avec Raylib et la création du projet de jeu.

Parcourir un vector 🔄

std::vector<int>  positions = {100, 200, 300};

// Avec for classique
for (int i = 0; i < positions.size(); i++) {
    printf("Position %d : %d\n", i, positions[i]);
}

// Avec range-based for (recommandé)
for (int pos : positions) {
    printf("Position : %d\n", pos);
}

Structures dans des Tableaux 🎯

Voici la partie la plus puissante : combiner structures et conteneurs ! Imaginons qu'on veuille stocker plusieurs voitures, chacune avec sa position, vitesse et couleur.
Cela combiner aux loop (boucles) vous avez la votre boucle de jeu.
Et pouvez faire vivre->bouger/interagir puis finalement mourrir ce petit monde. (bientôt le votre 😉)

Tableau de structures (array)

#include 

struct Car {
    int x;
    int speed;
};

int main() {
    // Tableau de 3 voitures
    Car cars[3] = {
        {100, 5},   // Voiture 0
        {300, 3},   // Voiture 1
        {500, 7}    // Voiture 2
    };
    
    // Accéder à la position X de la deuxième voiture (index 1)
    printf("Voiture 1 en position : %d\n", cars[1].x);
    
    // Modifier la vitesse de la première voiture (index 0)
    cars[0].speed = 10;
    
    // Parcourir toutes les voitures
    for (int i = 0; i < 3; i++) {
        printf("Voiture %d : position %d, vitesse %d\n", 
               i, cars[i].x, cars[i].speed);
    }
    
    return 0;
}

Syntaxe pour accéder à un membre d'une structure dans un tableau :

nomTableau[index].nomMembre

Exemple : cars[1].x accède au membre x de la structure à l'index 1

Vector de structures (std::vector)

#include 
#include 

struct Car {
    int x;
    int speed;
};

int main() {
    // Vector de voitures (vide au départ)
    std::vector cars;
    
    // Ajouter des voitures dynamiquement
    cars.push_back({100, 5});
    cars.push_back({300, 3});
    cars.push_back({500, 7});
    
    printf("Nombre de voitures : %zu\n", cars.size());
    
    // Accéder à un élément spécifique
    printf("Voiture 0 en position : %d\n", cars[0].x);
    
    // Modifier un membre
    cars[1].speed = 8;
    
    // Parcourir avec range-based for
    int index = 0;
    for (const Car& car : cars) {
        printf("Voiture %d : x=%d, speed=%d\n", 
               index++, car.x, car.speed);
    }
    
    // Supprimer la dernière voiture
    cars.pop_back();
    
    printf("Voitures restantes : %zu\n", cars.size());
    
    return 0;
}
Pourquoi const Car& dans la boucle ?
  • & : Évite de copier la structure (plus performant)
  • const : Indique qu'on ne va pas modifier l'élément
  • Si vous voulez modifier : utilisez juste Car& (sans const)

Les Maps : Tableaux Clé-Valeur 🗺️

Une std::map est un conteneur qui stocke des paires clé-valeur. Au lieu d'utiliser un index numérique (0, 1, 2...), vous utilisez une clé personnalisée (un nom, un ID, etc.) pour accéder à une valeur.

Analogie : C'est comme un dictionnaire ! Vous cherchez un mot (la clé) pour obtenir sa définition (la valeur).

std::map - Version triée 📚

La std::map garde les clés triées automatiquement grâce à une structure d'arbre binaire équilibré.
(syteme de gestion assez poussé mais pas impossible à comprendre si vous voulez faire vos recherche.) 

#include <map>
#include <cstdio>
#include <string>//il s'agit de la lib standard pour les chaines de caractère très utile.

int main() {
    // Map qui associe un nom de joueur à son score
    std::map<std::string, int> playerScores;
    
    // Ajouter des paires clé-valeur
    playerScores["Alice"] = 1500;
    playerScores["Bob"] = 2300;
    playerScores["Charlie"] = 1800;
    
    // Accéder à une valeur via sa clé
    printf("Score d'Alice : %d\n", playerScores["Alice"]);
    
    // Modifier une valeur
    playerScores["Bob"] = 2500;
    
    // Vérifier si une clé existe
    if (playerScores.find("David") != playerScores.end()) {
        printf("David trouvé !\n");
    } else {
        printf("David n'existe pas\n");
    }
    
    // Parcourir toutes les paires
    for (const auto& pair : playerScores) {
        printf("%s : %d points\n", pair.first.c_str(), pair.second);
    }
    
    return 0;
}

Structure d'une paire :

  • pair.first : La clé
  • pair.second : La valeur

std::unordered_map - Version non triée ⚡

La std::unordered_map fonctionne exactement comme std::map, mais ne trie pas les clés. En échange, elle est généralement plus rapide pour l'accès et l'insertion.

#include <unordered_map> 
#include <string> 
#include <cstdio> 

int main() {
    // Unordered_map pour stocker les vies par niveau
    std::unordered_map<int, int >  livesPerLevel;
    
    // Ajouter des données
    livesPerLevel[1] = 3;
    livesPerLevel[2] = 5;
    livesPerLevel[3] = 2;
    
    // Accès rapide
    printf("Vies au niveau 2 : %d\n", livesPerLevel[2]);
    
    // Parcours (ordre non garanti !)
    for (const auto& pair : livesPerLevel) {
        printf("Niveau %d : %d vies\n", pair.first, pair.second);
    }
    
    return 0;
}

Méthodes essentielles des Maps 🛠️

Méthode Description Exemple
[clé] Accéder ou créer une valeur map["key"] = 10;
find(clé) Chercher une clé map.find("key")
size() Nombre de paires int n = map.size();
erase(clé) Supprimer une paire map.erase("key");
clear() Vider la map map.clear();

Comparaison Map vs Unordered_map

Critère std::map std::unordered_map
Ordre ✅ Clés triées ❌ Pas d'ordre garanti
Performance accès O(log n) ⚡ O(1) en moyenne
Utilisation mémoire Modérée Plus élevée
Cas d'usage Besoin d'ordre trié Performance maximale

Quand utiliser quoi ?

  • 🗂️ std::map : Quand l'ordre des clés est important (leaderboard, données chronologiques)
  • ⚡ std::unordered_map : Quand seul l'accès rapide compte (cache, tables de hachage)

Partie Pratique

Exemple 1 : Système de score avec Map

#include <map> 
#include <string> 
#include <cstdio> 

int main() {
    std::map<std::string, int>  highScores;
    
    // Ajouter des scores
    highScores["Alice"] = 1500;
    highScores["Bob"] = 2300;
    highScores["Charlie"] = 1800;
    
    // Afficher le classement (automatiquement trié par nom)
    printf("=== CLASSEMENT ===\n");
    for (const auto& player : highScores) {
        printf("%s : %d points\n", 
               player.first.c_str(), player.second);
    }
    
    // Mettre à jour un score
    printf("\nBob améliore son score !\n");
    highScores["Bob"] += 500;
    
    printf("Nouveau score de Bob : %d\n", highScores["Bob"]);
    
    return 0;
}

Exemple 2 : Inventaire de jeu avec Unordered_map

#include <unordered_map>
#include <string>
#include <cstdio>

int main() {
    // Inventaire : nom d'objet → quantité
    std::unordered_map<std::string, int> inventory;
    
    // Ajouter des objets
    inventory["Potion"] = 5;
    inventory["Épée"] = 1;
    inventory["Bouclier"] = 1;
    
    // Utiliser une potion
    printf("Utilisation d'une potion...\n");
    inventory["Potion"]--;
    
    // Afficher l'inventaire
    printf("\n=== INVENTAIRE ===\n");
    for (const auto& item : inventory) {
        printf("%s x%d\n", item.first.c_str(), item.second);
    }
    
    // Vérifier si un objet existe
    if (inventory.find("Arc") != inventory.end()) {
        printf("\nVous avez un arc !\n");
    } else {
        printf("\nVous n'avez pas d'arc.\n");
    }
    
    return 0;
}

Conclusion : Choisir le bon conteneur 🎯

Le C++ offre une grande variété de conteneurs, chacun optimisé pour des situations spécifiques. Il n'existe pas de conteneur "parfait" qui convient à tous les cas !

Guide de sélection rapide

  • 📦 Array : Taille fixe connue, performance maximale
  • 🚀 Vector : Taille variable, ajout/suppression en fin de liste
  • 🗂️ Map : Accès par clé, besoin d'ordre trié
  • ⚡ Unordered_map : Accès par clé, performance maximale

Autres conteneurs C++ (vus ultérieurement) :

  • std::list : Insertion/suppression rapide n'importe où
  • std::set : Collection d'éléments uniques triés
  • std::queue : File d'attente (FIFO)
  • std::stack : Pile (LIFO)
  • std::deque : Insertion/suppression rapide aux deux extrémités

Compromis Performance vs Flexibilité ⚖️

Le choix d'un conteneur implique souvent des compromis :

Trade-offs classiques :
  • Vitesse vs Mémoire : Les conteneurs rapides (unordered_map) consomment plus de mémoire
  • Flexibilité vs Performance : Les tableaux statiques sont plus rapides mais moins flexibles que les vectors
  • Ordre vs Vitesse : Map maintient l'ordre mais est plus lent qu'unordered_map
  • Simplicité vs Optimisation : Parfois, un conteneur simple suffit même s'il n'est pas "optimal"

Règle d'or : Commencez avec le conteneur le plus simple qui résout votre problème (vector dans la plupart des cas). Optimisez seulement si vous rencontrez des problèmes de performance mesurables !

Résumé

À retenir

  • Index : Position d'un élément, commence TOUJOURS à 0
  • Array : Tableau de taille fixe, rapide, simple
  • Vector : Tableau dynamique, taille variable, plus flexible
  • Struct : Regroupe plusieurs données de types différents
  • Map : Conteneur clé-valeur trié
  • Unordered_map : Conteneur clé-valeur non trié
  • Accès tableau : tableau[index]
  • Accès struct : structure.membre
  • Struct dans tableau : tableau[index].membre
  • Accès map : map[clé] ou map.find(clé)

  • ✅ Déclarer et utiliser un std::vector
  • ✅ Accéder à un élément avec [index]

Points de vigilance :

  • Ne jamais dépasser la taille du tableau (index < taille)
  • L'index du dernier élément = taille - 1
  • Utiliser size() avec les vectors et maps, pas avec les arrays
  • Toujours initialiser les structures avant utilisation
  • Vérifier l'existence d'une clé avec find() avant d'y accéder
  • Utiliser const& dans les boucles pour éviter les copies inutiles

Définitions

Const-correctness : Le concept de code de qualité et de bonne pratique. Pas toujours évident à mettre en place, mais peut être peut on l'automatisé avec les IA de nos jours. Quoi qu'il en soit si on est en mesure de définir const toutes les variables qui le sont dans notre code montre une certaine maitrise du projet et du code en générale. Il est donc bon de s'exercer à cette pratique. Pour ma part, j'ai largement echouer jusqu'a maintenant. Mais il n'est jamais trop tard.

Conteneur (Container) : Structure de données qui stocke plusieurs éléments. Les tableaux, vectors et maps sont des conteneurs.

Index (Indice) : Position d'un élément dans un conteneur. Commence toujours à 0 en C++.

Array (Tableau) : Conteneur de taille fixe définie à la compilation. Rapide mais inflexible.

Vector : Conteneur de taille dynamique qui peut grandir ou rétrécir pendant l'exécution. Plus flexible mais légèrement plus lent.

Structure (struct) : Type de données personnalisé qui regroupe plusieurs membres de types différents sous un même nom.

Membre (Member) : Variable à l'intérieur d'une structure. On y accède avec l'opérateur point .

Map : Conteneur associatif qui stocke des paires clé-valeur triées. Utilise un arbre binaire équilibré.

Unordered_map : Conteneur associatif qui stocke des paires clé-valeur non triées. Utilise une table de hachage pour un accès rapide.

Paire (Pair) : Structure de deux éléments : first (la clé) et second (la valeur).

Clé (Key) : Identifiant unique utilisé pour accéder à une valeur dans une map.

Valeur (Value) : Donnée associée à une clé dans une map.
