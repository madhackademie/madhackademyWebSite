Struct & Methodes C++

GameDevReady

Objectif du Module

Jusqu'ici, une struct regroupe des données (HP, nom, puissance d'attaque…). Dans ce module, on y ajoute des méthodes : des fonctions déclarées à l'intérieur de la struct, qui agissent sur ses propres membres.

C'est le pont entre « j'ai des variables groupées » et le futur challenge David & Goliath : deux combattants, chacun avec sa struct, qui s'attaquent tour par tour — d'abord en console, puis affichés dans Raylib.

Ce que l'élève saura faire à la fin

  • Déclarer une struct avec des champs et des méthodes membres
  • Appeler une méthode sur une instance : david.Attack(goliath)
  • Afficher l'état du duel en console et dans la fenêtre Raylib
Prérequis : Print, Variables (struct simple), Conditions, Boucles, STD_Fonctions (fonction + retour struct).

Théorie

Le problème : une struct sans méthode, c'est un sac sans bras

Sans méthode, la struct contient des données, mais elle ne fait rien toute seule. Pour faire attaquer David, on doit écrire le calcul de dégâts en dehors, dans main(). Ça fonctionne… mais dès qu'on a 10 types d'ennemis, le code explose.

❌ Sans méthode

Les données et les actions sont séparées. Le main() doit tout gérer lui-même.

// Dans main() :
goliath.hp -= david.attackPower;
if (goliath.hp < 0) goliath.hp = 0;
printf("David attaque...\n");
➡️

✅ Avec méthode

L'action fait partie du combattant. Un seul appel depuis main().

// Dans main() :
david.Attack(goliath);

// La logique est DANS la struct.
En résumé : une méthode membre, c'est une fonction normale, mais rangée dans la struct. Elle a accès automatiquement à tous les champs de l'instance (name, hp, attackPower) sans qu'on ait besoin de les passer en paramètre.

Étape 1 — La struct de données (rappel)

On part de ce que vous connaissez déjà :

struct Fighter
{
    std::string name;       // le nom du combattant
    int hp;                // points de vie
    int attackPower;       // dégâts infligés par attaque
};

Pour l'instant, rien de nouveau. On va maintenant ajouter les méthodes à l'intérieur.

Étape 2 — La méthode Attack

Voici comment David peut attaquer Goliath. Le schéma d'abord, le code ensuite :

DAVID
attackPower = 8
→ appelle Attack()
⚔️ ──▶
target.hp -= attackPower
GOLIATH
hp : 50 → 42
reçoit la référence &
void Attack(Fighter& target) 1
{
    target.hp -= attackPower; 2
    if (target.hp < 0)
        target.hp = 0;         3
    printf("%s attaque %s pour %d degats !\n",
           name.c_str(), target.name.c_str(), attackPower); 4
}
  • 1 Fighter& target : on passe la cible par référence (&). Sans le &, on travaillerait sur une copie — les HP de Goliath ne changeraient pas !
  • 2 attackPower tout seul désigne le champ de l'attaquant (David). Pas besoin d'écrire david.attackPower — on est déjà à l'intérieur de la struct de David.
  • 3 On "clamp" les HP : on empêche de passer en dessous de 0 (impossible d'avoir −3 HP).
  • 4 name.c_str() : std::string n'est pas un char*, donc printf a besoin de cette conversion pour lire la chaîne. C'est une obligation technique de mélanger C et C++.

Étape 3 — Les méthodes de lecture : const

Certaines méthodes ne modifient pas les données du combattant — elles se contentent de les lire. On les marque const après les parenthèses. Cela dit au compilateur : "je promets de ne rien modifier".

bool IsAlive() const 1
{
    return hp > 0; 2
}

void PrintStatus() const 1
{
    printf("%s — HP: %d | ATK: %d\n", name.c_str(), hp, attackPower);
}
  • 1 Le mot-clé const après () signifie que cette méthode ne peut pas modifier hp, name ou attackPower. C'est une garantie de sécurité.
  • 2 hp > 0 est une expression qui retourne directement true ou false. On aurait pu écrire un if plus long — return est plus élégant.

Récapitulatif des 3 méthodes

Méthode Rôle Modifie des données ? Retourne
Attack(Fighter& target) Inflige des dégâts à la cible ✅ target.hp void (rien)
IsAlive() Vérifie si le combattant est encore en vie ❌ lecture seule bool (vrai/faux)
PrintStatus() Affiche le nom, les HP et l'ATK dans la console ❌ lecture seule void (rien)

La struct complète

Voici la struct avec les données et les méthodes réunies dans le même bloc :

struct Fighter
{
    // ── données ──────────────────────────────────────
    std::string name;
    int hp;
    int attackPower;

    // ── méthodes ─────────────────────────────────────
    void Attack(Fighter& target)
    {
        target.hp -= attackPower;
        if (target.hp < 0) target.hp = 0;
        printf("%s attaque %s pour %d degats !\n",
               name.c_str(), target.name.c_str(), attackPower);
    }

    bool IsAlive() const
    {
        return hp > 0;
    }

    void PrintStatus() const
    {
        printf("%s — HP: %d | ATK: %d\n", name.c_str(), hp, attackPower);
    }
};
⚠️ Pas encore de la POO classique : pas de class, pas d'héritage, pas de constructeur explicite. On reste sur une struct C++ enrichie — c'est suffisant, propre, et exactement ce qu'il faut pour le duel David & Goliath.

Étape 4 — Instancier et appeler depuis main()

Fighter david{"David", 30, 8};   1
Fighter goliath{"Goliath", 50, 12};

david.PrintStatus();   2
goliath.PrintStatus();

david.Attack(goliath);  3
goliath.Attack(david);

printf("David vivant : %s\n", david.IsAlive() ? "oui" : "non"); 4
  • 1 Initialisation entre {} : les valeurs sont assignées dans l'ordre des champs de la struct (name, hp, attackPower).
  • 2 On appelle la méthode avec le point . : david.PrintStatus(). C'est David qui exécute sa propre méthode — il a accès à son name, son hp, etc.
  • 3 david.Attack(goliath) : David est l'attaquant (on est dans sa méthode), Goliath est la cible (passé par référence &).
  • 4 L'opérateur ternaire ? : est un raccourci pour un if/else sur une ligne : si IsAlive() retourne true → "oui", sinon → "non".

Correction complète

Code de référence aligné sur 0x_Struct_Methodes/Correction/main.cpp :

#include "raylib.h"
#include <cstdio>
#include <string>

struct Fighter
{
    std::string name;
    int hp;
    int attackPower;

    void Attack(Fighter& target)
    {
        target.hp -= attackPower;
        if (target.hp < 0) target.hp = 0;
        printf("%s attaque %s pour %d degats !\n", name.c_str(), target.name.c_str(), attackPower);
    }

    bool IsAlive() const
    {
        return hp > 0;
    }

    void PrintStatus() const
    {
        printf("%s — HP: %d | ATK: %d\n", name.c_str(), hp, attackPower);
    }
};

int main(void)
{
    // instanciation de david et goliath
    Fighter david{"David", 30, 8};
    Fighter goliath{"Goliath", 50, 12};

    // état initial
    printf("--- Etat initial ---\n");
    david.PrintStatus();
    goliath.PrintStatus();

    // un échange d'attaques
    david.Attack(goliath);
    goliath.Attack(david);

    // état après l'échange
    printf("--- Apres un echange ---\n");
    david.PrintStatus();
    goliath.PrintStatus();

    InitWindow(800, 450, "GameDevReady — Struct & Methodes");
    while (!WindowShouldClose())
    {
        BeginDrawing();
        ClearBackground(RAYWHITE);
        // affichage des HP
        DrawText(TextFormat("%s  HP: %d", david.name.c_str(), david.hp),   50, 80,  28, BLUE);
        DrawText(TextFormat("%s  HP: %d", goliath.name.c_str(), goliath.hp), 50, 130, 28, MAROON);
        // état vivant/mort
        DrawText(TextFormat("David vivant: %s",   david.IsAlive()   ? "oui" : "non"), 50, 200, 22, DARKGRAY);
        DrawText(TextFormat("Goliath vivant: %s", goliath.IsAlive() ? "oui" : "non"), 50, 230, 22, DARKGRAY);
        EndDrawing();
    }
    CloseWindow();

    return 0;
}

Résumé

À retenir

  • Une méthode membre est une fonction rangée dans la struct — elle voit tous ses champs directement
  • Fighter& target — passer par référence (&) pour modifier la vraie cible, pas une copie
  • const après () = méthode de lecture seule — le compilateur empêche toute modification accidentelle
  • instance.Methode(autreInstance) — la syntaxe avec le point . pour appeler
  • Console + Raylib : même logique, deux sorties visuelles différentes
