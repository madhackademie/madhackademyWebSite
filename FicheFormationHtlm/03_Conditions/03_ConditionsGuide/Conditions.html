Conditions C++

GameDevReady

Objectif du Module

Les conditions sont essentielles en programmation. Il y a 4 grandes structures conditionnelles :

  • if (si) - la condition de base
  • else (sinon)
  • else if (sinon si)
  • switch (une structure qui remplace une suite de else if) - sera vue plus tard en refactorisation

Points clés de ce module

  • Les structures de contrôle (ou instructions conditionnelles)
  • L'indentation
  • La portée des variables

Pour les conditions, prenons la cuisson des pâtes comme un programme. Dans ce programme nous avons : si l'eau bout alors mettre les pâtes. Il s'agit là d'une instruction conditionnelle. Indispensable en programmation, il s'agit de toute la logique nécessaire pour résoudre votre problématique et transformer A en B si c'est votre objectif.

L'indentation nous permet d'avoir une structure ou une architecture visuelle de cette logique et elle délimite aussi la portée des variables. Disons, en essayant de simplifier, qu'une variable peut être existante pour tout le programme ou seulement une partie de ce dernier. Elles sont définies de la manière qui suit :

  • Globale (tout le programme)
  • Locale (au sein d'une fonction par exemple)

If Else - Avec alternative

Passons à une option un peu plus avancée qui va nous permettre de voir le principe des blocs de code et/ou la portée.

IMAGE : Exemple de condition if else

On peut le traduire ainsi : Si (age supérieur ou égal à 18) {fait quelque chose} sinon {fait autre chose}. Encore une fois il y a les balises de couleurs : 32 vert et 31 rouge .

Information importante

Je crois que je m'avance un peu mais avec cette condition vous pouvez tout coder (Wow, Windows, la nouvelle tech IA de demain).

Ici une nouveauté importante : les { } accolades ou brackets délimitent le bloc de code. Si on remplit la condition, alors ce qui se trouve entre ces accolades sera interprété.

La Portée des Variables

IMAGE : Portée globale vs locale & Erreur de portée de variable

Ici on peut voir une variable globale age (encadré en vert) qui est accessible partout dans le code du Main. Alors que la variable test (encadré en rouge) n'est accessible qu'entre les { } où elle a été initialisée. On pourrait l'appeler une variable poubelle ou éphémère. Les { } nous permettent aussi de mettre en place ce que l'on appelle l'indentation, et donc de voir visuellement les blocs de code imbriqués les uns dans les autres.

Si une variable est créée dans un bloc, elle n'existera que dans cet espace et pas en dehors du bloc. Il s'agit de la différence entre une variable globale et locale. On peux voir que, j'ai codé une erreur liée à la portée d'une variable qui n'est pas dans le "scope" (portée en anglais).

Exercices

Exercice principal

Ici vous allez devoir créer une variable int (pour un contrôle d'âge) d'une valeur de 10. Ensuite vous allez devoir afficher à l'écran de jeu :

  • En rouge : un message si l'utilisateur n'a pas donné de valeur valide (0 ou moins) : "Veuillez donner un âge valide !"
  • En vert : "Super vous êtes vivant !"

Le challenge

Le challenge est double :

  1. Passer la couleur en variable pour apprendre à fouiller dans raylib et découvrir l'une des premières variables nécessaires de la lib.
  2. Déterminer si l'âge est == avec un print en bleu : "Oh vous avez exactement x ans, vous gagnez le premier prix mystère !" (il y a 2 manières de procéder à ma connaissance)

Essayez de ne pas foncer sur la solution. Le travail de recherche est celui qui est le plus important, même s'il est fait avec une IA.

Jouez avec la valeur intégrale pour vérifier si toutes les conditions fonctionnent.

Résumé

Points Clés

  • Point clé 1 : Les conditions if, else, else if contrôlent le flux du programme.
  • Point clé 2 : Les accolades { } délimitent les blocs de code et la portée des variables.
  • Point clé 3 : L'indentation rend le code lisible et est gérée automatiquement par l'éditeur.

À retenir

  • Règle importante : Toujours indenter correctement son code.
  • Piège à éviter : Utiliser une variable hors de sa portée (scope).
  • Bonne pratique : Valider les valeurs avant de les traiter (âge >= 0, etc.).

Définitions

Refactorisation : Uncle Bob va vous aimer ! Il s'agit de corriger ou améliorer un code déjà existant de manière à éliminer un "bug" ou rendre le code plus lisible, comme renommer une variable de manière plus explicite. Ou encore de transformer une série de else if en un switch (vu plus tard). C'est une méthodologie qui va de pair avec le code smell (qui lui vous dira de changer les switch par autre chose 😊). Elles font toutes deux partie d'une philosophie nommée Clean Code (encore une fois Uncle Bob).

Balises : Une balise est un élément de code qui délimite ou marque un contenu afin que le programme ou le navigateur sache comment l'interpréter ou l'afficher.

Clean Code : Philosophie, méthodologie pour écrire un code soi-disant de manière propre, maintenable et évoluable. Qui définit le principe SOLID. Oui, dans cette sphère de la programmation il y a beaucoup de choses qui font toujours encore débat mais qui sont des principes méthodologiques fonctionnels dans la majeure partie des situations. Mais en fonction du paradigme de code que vous allez choisir toutes ces informations seront peut-être inutiles.

Console : Élément de l'éditeur qui essaye de communiquer avec nous tant bien que mal.

Main : Il s'agit du code central ou principal vous en apprendrez plus lors du nettoyage du code

Indentation : C'est comme le sommaire d'un livre ou une liste à puces : cela permet de voir visuellement la hiérarchie de ton code.
